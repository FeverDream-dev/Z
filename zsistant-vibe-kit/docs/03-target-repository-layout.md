# 03 — Target Repository Layout

This file describes the structure OpenCode should eventually create. This kit does not include the source files yet.

```text
zsistant/
  README.md
  AGENTS.md
  LICENSE
  docs/
  assets/
  prompts/
  personas/
  phases/
  cmd/zazi/                  # generated by OpenCode later
  internal/                  # generated by OpenCode later
    config/
    daemon/
    agents/
    store/
    jobs/
    llm/
    channels/
    bus/
    trainer/
    skills/
    sandbox/
    server/
  web/                       # generated by OpenCode later if UI build is needed
  tests/
  .github/workflows/
```

## Source code generation policy

OpenCode should create source code only when the human starts a phase prompt.

## Runtime data layout

Runtime data should not live inside the repo by default.

Recommended default:

```text
~/.zazi/
  config.yaml
  agents/
  logs/
  cache/
  backups/
```

## Agent data layout

```text
~/.zazi/agents/<agent-id>/
  profile.json
  persona.md
  permissions.json
  channels.json
  memory/
  workspace/
  jobs/
  audit.jsonl
```

## Naming conventions

- Product: Zsistant.
- CLI: `zazi`.
- Daemon: `zazi daemon` or `zazi serve`.
- Web UI title: Zsistant Control Tower.
- Default coding agent: Z Assistant.
