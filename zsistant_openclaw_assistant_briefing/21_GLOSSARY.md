# Glossary

## Assistant

A persistent AI actor with identity, purpose, persona, tools, memory, knowledge, channels, jobs, permissions, and logs.

## Chat

A conversation surface for interacting with an assistant. Chat is not the whole product.

## Channel

A communication surface where messages can enter or leave, such as Web UI, Telegram, WhatsApp, Slack, Discord, CLI, email, or voice.

## Tool

A capability the assistant can use to observe or act, such as browser control, search, files, email, calendar, or an MCP server.

## MCP

A tool/server integration pattern that exposes external capabilities to the assistant.

## Browser/Chrome tool

A tool that lets the assistant open pages, inspect UI, capture screenshots, and perform browser actions when permitted.

## Persona

The assistant's behavior style, tone, role, and boundaries.

## Knowledge

Documents, files, notes, project context, webpages, or references the assistant can use.

## Memory

Durable learned context saved for future use. Memory should be inspectable and editable.

## Job

Scheduled work owned by an assistant.

## Automation

Event-triggered work owned by an assistant or workspace.

## Permission

A rule controlling what an assistant can read or do.

## Approval

A user decision required before a sensitive action proceeds.

## Developer Mode

An advanced UI mode that exposes traces, raw messages, provider routing, tool calls, MCP status, browser actions, and diagnostics.

## Gateway

A routing/control layer that connects channels, assistants, tools, and sessions.

## Surface

Any user-facing interaction point, such as Web UI, CLI, Telegram, or mobile.

## Activity timeline

A chronological record of assistant actions, messages, jobs, errors, approvals, and tool calls.
