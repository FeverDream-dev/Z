# Public OpenClaw Source Notes

These notes summarize public material checked on 2026-05-03. They are included so the coding AI understands that OpenClaw-like assistants are not merely chat pages.

## Official docs

Source: https://docs.openclaw.ai/

Key product ideas:

- OpenClaw is described as an any-OS gateway for AI agents across many communication channels.
- Channels mentioned include Discord, Google Chat, iMessage, Matrix, Microsoft Teams, Signal, Slack, Telegram, WhatsApp, Zalo, and more.
- The docs describe OpenClaw as a self-hosted gateway connecting chat/channel surfaces to AI coding agents.
- The docs emphasize a single gateway as the source of truth for sessions, routing, and channel connections.
- Capabilities include multi-channel gateway, plugin channels, multi-agent routing, media support, Web Control UI, and mobile nodes.

## Official GitHub README

Source: https://github.com/openclaw/openclaw

Key product ideas:

- OpenClaw is described as a personal AI assistant that runs on the user's own devices.
- It answers on channels the user already uses.
- It can speak/listen on desktop/mobile platforms and render a live Canvas.
- The README says the Gateway is the control plane, but the product is the assistant.
- Supported channels include WhatsApp, Telegram, Slack, Discord, Google Chat, Signal, iMessage, Microsoft Teams, Matrix, LINE, Mattermost, Twitch, Zalo, WebChat, and more.
- The README discusses multi-agent routing, voice/talk modes, live canvas, local-first gateway, security defaults, and sandboxing.

## Official website

Source: https://openclaw.ai/

Key product ideas:

- The public headline is "The AI that actually does things."
- Examples include clearing inboxes, sending emails, managing calendars, and checking in for flights.
- Public testimonials emphasize persistent memory, persona onboarding, communications integration, heartbeats, proactive cron jobs, reminders, background tasks, and assistants with access to a computer.

## Interpretation for Zsistant

The lesson is not to copy OpenClaw line-by-line.

The lesson is that the product category is assistant platform, not chat UI.

Zsistant should therefore emphasize:

- Assistant identity
- Multi-channel access
- Gateway/routing thinking
- Tools and actions
- Browser/computer access
- Jobs and schedules
- Memory and knowledge
- Multi-agent routing
- Permissions and safety
- Developer observability
